# Ruthless review and iteration

## V1 - reuse the previous raw-asset list

Rejected.

The prior list included command palettes, settings, Smart Context handoff, Dedupe progress/results, and generic Pro hero art. Those assets may be useful for documentation or suite storytelling, but they violate the current screenshot campaign's strongest conversion sequence and the cut list.

## V2 - use the existing five-image campaign directly

Rejected.

The mixed sequence included Smart Lookup as image 3. The current product boundary identifies Smart Lookup as a standalone plugin, so it cannot fill a Smart Connections-only AG1 proof slot. It also did not yield enough distinct real UI evidence for both AG1 and AG3.

## V3 - capture one wide screenshot per scene and crop all formats

Rejected.

A portrait crop from a wide desktop state can turn the UI into unreadable texture or remove the source/output relationship. The SOP requires publication-size readability. Separate aspect-specific raw captures are more reliable.

## V4 - direct official capture from draft scenes

Rejected.

The manifest marks every supplied scene as `draft`. Skipping verification would violate the lifecycle rule that only `capture-ready` scenes may be delegated for official capture.

## V5 - final architecture

Accepted.

- two-stage verification then official capture
- AG1 uses only Smart Connections Core
- AG3 uses only Connections Pro depth
- no Lookup, Context, Dedupe, settings, errors, progress, or command-list imagery in the primary set
- four evidence families per asset group
- three separately composed aspect variants per family
- 24 final files with exact PMax dimensions
- two new scene notes close the actual control-point gaps: Footer Core and Bases related-link trails

## marginal-improvement boundary

Further iteration now changes taste more than truth or operating reliability.

The next meaningful improvement can come only from observing the current UI during verification and adjusting exact labels, layout, and result contracts to the installed build.
