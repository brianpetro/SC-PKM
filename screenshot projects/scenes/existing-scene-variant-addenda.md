## `Smart Connections scene - 01 current note related notes.md`

Replace the `## variants` table with:

```md
## variants

| Variant ID | What changes | What remains invariant | Eligible asset classes |
|---|---|---|---|
| `desktop-sources` | desktop Connections pane | active note causes ranked related notes | all |
| `expanded-result` | one verified source result is expanded in the list | active note remains the anchor and expanded content is real | conversion, feature docs |
| `footer-core` | Footer connections replaces sidebar | current note causes related notes | feature/docs; use the separate verified Footer scene |
| `motion-updates` | anchor prose changes while list updates | list visibly follows current note | editorial/docs motion |
```

Add this result contract for `expanded-result`:

```md
- The expanded result belongs to a target note already present in the current list.
- The visible snippet matches the real fixture note.
- The active anchor remains visible.
- At least one neighboring ranked result remains visible.
- No hover preview, menu, or transient control obscures the result.
```
