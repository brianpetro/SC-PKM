# Smart Connections - screenshot campaign

## executive decision

Final product screenshots must be real captures from the current Obsidian UI.

Generated images may be used as storyboards or external editorial treatment. They must not be presented as product UI.

The intended ecosystem story is:

```md
Current note -> related notes -> useful link -> question-first Lookup -> Pro adds in-editor and Bases depth.
```

## current product-boundary correction

Smart Lookup is a standalone Core plugin in the current product architecture.

Therefore:

- Scene 03 is valid for Smart Plugins ecosystem pages and documentation.
- Scene 03 must be labeled `Smart Lookup`, not presented as a Smart Connections Core surface.
- Scene 03 is not eligible for a Smart Connections-only plugin-store image set.
- A Smart Connections-only listing should replace Scene 03 with a verified Core scene, preferably Footer connections.

Do not use the retired filename:

```text
smart-connections-03-lookup-question-by-meaning.png
```

Use:

```text
smart-lookup-03-question-by-meaning.png
```

## campaign tracks

### Track A - Smart Plugins ecosystem story

Use Scenes 01-05 in order.

Scene 06 is an optional continuation.

This track truthfully crosses Smart Connections, Smart Lookup, Connections Pro, and Smart Context.

### Track B - Smart Connections-only plugin listing

Current recommended structure:

```md
01 Smart Connections Core: current note -> related notes
02 Smart Connections Core: useful result -> note link
03 replacement required: Footer connections Core
04 Connections Pro: inline connections
05 Connections Pro: Bases relevance
```

Do not publish Track B as a complete set until the Footer scene is specified, verified, and captured.

## campaign WDLL

- The first two images prove the flagship Smart Connections Core loop.
- Every image has one job, one visible outcome, and one caption.
- The same fixture topic and note titles create continuity across scenes.
- Pro appears only after the Core loop is already valuable.
- Pro screenshots feel additive rather than required to make Core useful.
- Cross-plugin scenes name the destination plugin explicitly.
- No static caption claims an interaction that only motion can prove.
- The ordered images and captions communicate the intended story without additional explanation.

## non-negotiable campaign rules

| Rule | Standard |
|---|---|
| Real UI only | Product UI and output originate in the current plugins. |
| One job per image | Each image proves exactly one user-visible outcome. |
| Same fixture | Reuse [[Smart Connections - screenshot fixture]] and its stable note family. |
| No setup anxiety in conversion sequence | Do not show indexing progress, threshold tuning, command lists, or settings. |
| Correct ownership | Smart Lookup and Smart Context are named as separate plugins when shown. |
| No accidental Pro in Core | Use isolated Core profiles; do not depend on cropping or DOM removal. |
| Pro is additive | Core value is established before Pro scenes. |
| Real output | Scores, ordering, snippets, links, and transferred items are not fabricated. |
| Proof-matched captions | Still captions describe visible state; interaction claims use motion. |
| No Dedupe in the primary set | Dedupe changes the first impression from discovery to cleanup. |

## campaign claims by scene

| Scene | The image may claim | The image must not imply |
|---|---|---|
| 01 | current-note-first related-note discovery | exact search, automatic truth, or Pro-only ranking behavior |
| 02 | a result became a real note link | drag unless the asset shows motion |
| 03 | Smart Lookup retrieves by meaning from a question | that Lookup is currently part of Smart Connections Core |
| 04 | Connections Pro surfaces matches for a block in the editor | that the popover is Core or a separate dashboard |
| 05 | Connections Pro adds and sorts a score relative to a reference note | that scores are universal grades or identical across models; the reference must be visible or established by a motion/pair asset |
| 06 | Connections results can populate Smart Context | that Smart Context is built into Smart Connections or that content was sent to an AI provider; a single still is eligible only when matching source results remain visible |

## cut list for the primary conversion sequence

| Cut | WHY |
|---|---|
| Duplicate Detector or Dedupe | Repositions the product as cleanup software. |
| Dedupe threshold modal | Looks like setup friction. |
| Scan/index progress | Shows waiting instead of value. |
| Settings filters | Better for procedural docs after first value. |
| Ranking algorithms | Too technical for the first impression. |
| Commands/ribbon view | Useful for getting-started docs, not conversion proof. |
| Codeblock UI | Too specialized for the primary story. |
| Error screenshots | Do not use in conversion assets. |
| Full Smart Context builder before Scene 06 | Introduces another product before the core loop is understood. |
| Generated or reconstructed UI | Breaks product trust. |

```md
1. Related notes appear for the note I am working on.
2. A useful result can become structure inside that note.

```
