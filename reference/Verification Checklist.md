> [!summary]
> The vault should be trustworthy. Verification is how you keep it that way.

> [!When to use]
> After using AI to generate notes, summaries, or drafts, especially when accuracy matters.

### Checklist
> [!summary]
> A quick checklist to prevent unverified claims from entering your vault.

## Verify before you keep
- [ ] Source: can I point to where this came from?
- [ ] Quote check: does the source actually say this (in context)?
- [ ] Numbers/dates/names: double-check exact values.
- [ ] Confidence: mark anything uncertain as uncertain.
- [ ] Rewrite: put the final note in your own words.

## If there is no source
- Treat it as a hypothesis.
- Add a question: "What evidence would confirm this?"

## For outputs
- Link key claims to the note or source they came from.

### Failure mode and fix
Failure mode: You skip verification because it feels slow.

Fix: verify only the load-bearing claims (numbers, dates, names, causal claims). Timebox to 3 minutes.

### Covers / Does not cover
- Covers: Verifying claims before storing them as truth.
- Does not cover: Academic-level citation standards.

## Next action
- Pick one AI-assisted note and run this checklist. Fix or mark any uncertain claim.
