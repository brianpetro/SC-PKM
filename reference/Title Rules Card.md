### Rules
# Title Rules Card

> [!summary]
> Titles are retrieval handles. Write them like future-you is searching.

## The 5 rules
1) Title should match the question you will search later.
2) Prefer concrete nouns and verbs over vague labels.
3) If it is a workflow, include "workflow" in the title.
4) If there is ambiguity, add a qualifier (PKM vs email, personal vs work).
5) Put synonyms in aliases, not in the title.

## Quick rewrites
- "Thoughts on productivity" -> "How to choose the next action when overwhelmed"
- "Meeting notes" -> "Meeting Notes Workflow"
- "Inbox" -> "PKM Inbox" or "Email Inbox"

## Stop rule
Rename only when retrieval pain repeats 2+ times.

### Failure mode and fix
Failure mode: You rename constantly and break your mental map.

Fix: rename only when retrieval pain repeats, and add the old title as an alias.
