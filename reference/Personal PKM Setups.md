> [!summary]
> Examples of vault structures tailored to different roles and workflows.

> [!When to use]
> If you are unsure what a vault should look like, this note shows real setup patterns by role, without copying someone else's system blindly.

### Setup patterns by persona and use case
- PARA-first: folder buckets for actionability.
- MOC-first (LYT-style): navigation through MOCs and contextual links.
- Most setups converge on light folders, a few MOCs, and a stable tag canon (see [[Personalized PKM System]] for the why).

### Folder, tag, and MOC structure examples
- One /references folder (notes)
- One /inbox folder (captures)
- MOCs as entry points (topics, projects, roles)

### Tool stack variants and tradeoffs
- Obsidian (notes + links + search)
- Simple sync + backup
- Capture tool on phone

### Migration from messy to structured setup
- Fix the worst naming issues.
- Create an inbox.
- Create 2-3 entry MOCs.

### How to choose and iterate your setup
- Your main outputs (study, research, writing, work execution).
- Your time budget.
- Your tolerance for structure.
