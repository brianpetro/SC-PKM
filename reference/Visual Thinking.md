> [!summary]
> Use diagrams to externalize concepts, see patterns, and improve understanding.

> [!When to use]
> If text notes hide structure, this note shows how to use simple visuals to reveal patterns, even if you do not think visually.

### Why visuals improve understanding and recall
- A topic has many moving parts.
- You need to understand relationships (cause/effect, sequence, dependencies).
- You are synthesizing across multiple notes.

### Diagram types to use (maps, flows, models)
- Mind map: explore a topic quickly (see [[Mind Mapping]]).
- Concept map: show labeled relationships between ideas.
- Flowchart: represent a process or workflow.

### Convert visuals into linked atomic notes
- The vault only improves if visuals become notes you can retrieve.
- Conversion pattern: 1) Draw the diagram (paper, Excalidraw, Mermaid).
- 2) Identify the 3-7 core nodes.

### Tool suggestions and export considerations
- Paper + photo (fastest).
- Excalidraw for sketching inside Obsidian.
- Mermaid for simple, portable diagrams in Markdown.

### Avoid pretty diagrams without meaning
- Fix: end every diagram session with one of:
- Fix: only draw when it reveals structure you did not have before.
- Symptom: you draw, feel smart, and never use the result.
