> [!summary]
> Use folders, tags, links, and PARA to keep structure flexible and findable.

> [!When to use]
> If folders, tags, and links feel like competing systems, this note shows how to combine them so organization stays flexible, without over-structuring.

### Hierarchy vs networked organization
- Synthesis and insight (see [[Networked Thought]]).
- Hierarchy (folders) makes scanning easy.
- Networks (links) make meaning and retrieval powerful.

### Folders vs tags tradeoffs and combos
- Define where something lives.
- Work well for Projects/Areas/Archives.
- Tend to be stable.

### PARA as a high-level filing lens
- Projects: active deliverables with an end.
- Areas: ongoing responsibilities.
- References: reference material and reusable knowledge.

### Linking and MOCs for navigation
- Create entry points into clusters.
- Avoid duplicating content.
- Keep a map for each domain or project.

### Rules to keep structure lightweight
- Fix: reorganize only to support retrieval or a real output.
- Keep folder depth shallow (0-2 levels).
- Prefer adding a link sentence over inventing a new tag.
