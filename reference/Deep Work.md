> [!summary]
> Deep work is protected time for cognitively demanding work that creates real output.

> [!When to use]
> When your best work keeps getting fragmented by meetings, notifications, or constant context switching.

### The rule
- Schedule a block.
- Choose one outcome.
- Remove distractions.
- Work until the block ends.
- Ship something small.

### A 5-step deep work setup
1) **Name the outcome** in one sentence (not a task list).
2) **Make the first move** obvious (open the file, outline 3 bullets).
3) **Lock distractions** (phone out of reach, notifications off, close extra tabs).
4) **Timebox** (45-90 minutes is plenty).
5) **End with a checkpoint** (what changed, what is next).

### Concrete example
Outcome: "Draft the intro + outline for the Starter Vault marketing page."

First move: Open the draft note and write 3 bullets: audience, promise, proof.

Timebox: 50 minutes.

Checkpoint: 1 paragraph drafted + next bullet list for section 2.

### Failure mode and fix
Failure mode: You start the block with vague intent ("work on X"), then drift.

Fix: rewrite the block title as a deliverable: "Ship draft paragraph" or "Decide between option A/B".
