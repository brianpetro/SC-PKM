> [!summary]
> Make relationships explicit so ideas gain meaning from how they connect.

> [!When to use]
> If ideas sit next to each other but never combine, this note shows how to write relationship sentences that create meaning, without over-linking.
> If your links are vague, this note gives you a small set of relationship labels so links carry meaning, without adding heavy metadata.

### Why concept relationships create insight
- meaning: why the concept matters,
- constraint: when it applies,
- transfer: how it generalizes to new domains.

### Why generic links are not enough
- Link spam: too many links with no meaning.
- Lost context: future you cannot remember why the link was added.
- Weak synthesis: you cannot quickly see support vs contradiction vs prerequisites.

### Relationship patterns to capture (supports, contrasts)
- example of: concrete instance of an abstract idea.
- supports: strengthens or provides evidence.
- contrasts: shows an alternative or different approach.
- example of: concrete instance of a general idea.
- supports: this note strengthens or provides evidence.
- contradicts: this note challenges or provides counter-evidence.

### How to write relationship sentences in notes
- Include a short example if possible.
- Adding links with no explanation. Fix: add one relationship sentence to the most important links, especially in evergreen notes.
- [[Progressive Summarization]] supports [[Second Brain]] because it turns raw capture into distilled, reusable material.

### Where to annotate the relationship (inline vs metadata)
- Pros: readable, meaningful, works everywhere.
- Cons: requires a sentence, not just a link.
- Pros: queryable (in theory).

### Cluster and map concepts into structures
- MOCs / index notes for a concept domain (see [[Index Notes and MOCs]]).
- Mind maps for exploration.
- Comparison tables for alternatives.

### Exercises to strengthen relational thinking
- Bridge exercise: pick two notes from different clusters and write one "bridge" paragraph linking them.
- Contrast exercise: write one note that contrasts two approaches and lists tradeoffs.
- Dependency chain: write a 5-step prerequisite chain for a workflow.

### Guardrails to keep it lightweight
- Do not label every link. Label the non-obvious ones.
- Use one relationship sentence per important link.
- Avoid inventing new relationship words for each note.
