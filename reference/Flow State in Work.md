> [!summary]
> Flow is more likely when the task is clear, challenging, and distraction-free.

> [!When to use]
> When you want deep focus but you keep bouncing between apps or feeling stuck at the start.

### 3 conditions that raise flow odds
- **Clear goal**: you know what done looks like for this session.
- **Immediate feedback**: you can tell if you are making progress.
- **No interruptions**: you can stay in one mental context long enough.

### How to tune challenge vs skill
- Too easy -> boredom.
- Too hard -> anxiety.

Fix: adjust the task so it is "stretching but doable".
- Reduce scope (one section, not the whole doc).
- Add structure (outline first).
- Add a constraint (timebox, word count, test to pass).

### Concrete example
Instead of "write the whole guide", do: "Write the 7-bullet outline and one example section in 30 minutes".

That creates clarity + feedback + a manageable challenge.

### Failure mode and fix
Failure mode: You try to enter flow while leaving open loops (messages, tabs, notifications).

Fix: run [[Pre-Flow Checklist]] first, then start the timer.
