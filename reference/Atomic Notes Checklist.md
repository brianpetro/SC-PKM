> [!summary]
> Quick audit to confirm a note is atomic, self-contained, and link-ready.

> [!When to use]
> Not sure if a note is actually reusable? This checklist lets you spot non-atomic notes fast and fix them, even when you only have a few minutes.

### Scope test for one idea per note
- [ ] The note has one primary claim, concept, or procedure.
- [ ] If there are multiple ideas, each could be moved to its own note without breaking meaning.
- [ ] The note is not secretly a log (meeting, journal, reading) that should be processed first.

### Standalone context and definitions check
- [ ] The note includes at least one concrete example.
- [ ] The first paragraph states what this note is about in plain language.
- [ ] Key terms are defined or linked to definitions.

### Title and alias hygiene check
- [ ] Title is specific and search-friendly (not "Notes on X").
- [ ] Title matches the question you expect to search later.
- [ ] Aliases include only realistic search phrases, not keyword soup.

### Link targets and relationship clarity check
- [ ] Each link earns its place with a relationship sentence (supports, contrasts, example of, prerequisite for).
- [ ] The note links to 1-3 truly related notes (not everything).
- [ ] There are no "naked link blocks" without context.

### Refactor actions and acceptance criteria
- 1 example and 1 link that clarifies meaning.
- Split: extract one idea into its own note.
- Add context: add 2-3 sentences so the note stands alone.

### Concrete example (apply the checklist in 3 minutes)
- Add one example sentence to each.
- Scope test fails (multiple ideas). Split into:
- "Hotkeys reduce capture friction"

### Failure mode
- Failure mode: treating the checklist as a perfection gate.
- Fix: timebox the checklist to 3 minutes per note, then move on.
- Use [[Note Refactoring and Maintenance]] during reviews for deeper cleanup.
