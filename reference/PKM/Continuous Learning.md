### Learning loop from input to application
- Continuous learning is not "consuming content".
- It is a loop: Input -> capture -> distill -> connect -> practice -> output -> review -> repeat The loop fails when you.
- The loop compounds when you consistently convert learning into something you can reuse or apply.

### Daily capture and quick reflection habit
- Capture: 1-3 things that mattered.
- Clarify: write one sentence in your own words.
- Connect: add one link to a related note.

### Review and spaced reinforcement practices
- Fix: reinforce only what you want to become behavior.
- Use spaced repetition for high-value knowledge (see [[Spaced Repetition]]).
- Review notes during a weekly/monthly cadence.

### Linking for cross-topic accumulation
- Add a relationship sentence: "This supports/contrasts with X because Y".
- Maintain a small MOC for your main learning domains.
- Create "bridge notes" that connect concepts from different areas.

### Output as proof of learning
- A decision you make differently.
- A paragraph you write.
- A checklist you add to a workflow.

# Continuous Learning

PKM supports continuous learning by providing a structured approach to capturing, organizing, and applying knowledge. This ensures that learning is a lifelong process, not just a one-time event. Key practices include:

1. **Daily Knowledge Intake**: Dedicate time each day to learn something new, whether it's reading, listening, or watching educational content.
2. **Regularly Updating Notes**: Consistently update your notes with new information and insights.
3. **Application of Knowledge**: Apply what you've learned in real-life scenarios to reinforce understanding.
4. **Feedback Loop**: Create a feedback loop by revisiting and refining your knowledge based on new experiences.
