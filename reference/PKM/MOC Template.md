> [!summary]
> Use this template to create a small hub note that points to the right notes with just enough glue text.

> [!When to use]
> When a topic is scattered across many notes and you want a single entry point that you can scan in under a minute.

### Template
Copy this into a new note and customize the links.

```md
> [!summary]
> What this MOC is for (1 sentence).

> [!When to use]
> The situation where you open this note.

## Start here
- [[Personal Knowledge Management Overview]]

## Core notes
- [[Capturing Information]] - capture boundary
- [[Processing and Organizing Notes]] - triage into usable artifacts
- [[Efficient Retrieval]] - make notes findable

## Workflows
- [[PKM Daily Workflow]]
- [[PKM Weekly Review]]
- [[Synthesis Workflow]]

## Templates and artifacts
- [[Vault Templates Library]]

## Common pitfalls
- Becoming a dumping list (keep it curated).
- Duplicating content from the linked notes.

## Next action
- Add 3 links you will actually use this week.
```

### Rules that keep MOCs useful
- Link out; do not rewrite atomic notes.
- Add one sentence of glue only when it clarifies why the link is here.
- Keep lists short enough to scan quickly (aim: 5-15 links total).
- Add a refresh cadence if the topic changes often (see [[MOC Refresh Checklist]]).
