> A single trusted place for knowledge so review, linking, and retrieval stay reliable.

> [!When to use]
> When notes live everywhere and you stop trusting them, this note helps you centralize so review and retrieval actually work, even across multiple devices.

### Why centralization reduces friction and loss
- Fewer capture drop-offs because there is only one destination.
- Better retrieval because search covers the whole knowledge base.
- Less duplication because you stop recreating the same notes.

### Define boundaries for what goes in/out
- Notes you want to retrieve later (decisions, procedures, insights, references).
- Work artifacts that benefit from linking (meeting outcomes, project context).
- Evergreen concepts you will reuse.

### Storage approach (local, sync, backups)
- Local-first files are the safest default (see [[Local First File Over App]]).
- Sync is convenience, not backup.
- Backups should be automated and tested.

### Navigation entry points (indexes, dashboards)
- A small set of index notes (see [[Index Notes and MOCs]]) for areas that matter.
- 1 inbox note or folder.
- Saved searches for review queues (see [[Queries and Saved Searches]]).

### Operational rules to keep trust high
- Failure mode: "Multiple inboxes."
- Failure mode: "Unreliable backups."
- Failure mode: "Sensitive content shared accidentally."

A centralized repository is crucial for a well-organized PKM system. It ensures that all knowledge is stored in a single, easily accessible location. Key practices include:

1. **Single Source of Truth**: Maintain one primary location for all knowledge storage.
2. **Cloud Storage**: Use cloud services like Google Drive, Dropbox, or OneDrive for access anywhere.
3. **Backup**: Regularly back up your repository to prevent data loss.

A strong PKM system has one core principle: everything goes into one trusted place.

### 🧠 Why Centralization Matters
- Reduces scattered information and tool fatigue
- Makes reviewing, linking, and retrieval easier
- Encourages regular usage and review

### 🛠️ How to Build One
1. **Choose your hub**: Obsidian, Notion, markdown folder — just one.
2. **Set clear boundaries**: What goes in, what stays out (no random screenshots?)
3. **Use a triage flow**: [[Capture Tools]] → [[PKM Daily Review]] → Permanent notes
