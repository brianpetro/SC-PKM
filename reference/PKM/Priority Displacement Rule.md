> [!summary]
> If you do not name the trade, you will pay it later.

> [!When to use]
> When new requests arrive and your current commitments are already full.

### Rule card
> [!summary]
> Every yes is a trade. Name the trade before you commit.

## The rule
Before you say yes, answer:
- What will this displace?
- What will be delayed?
- What standard will drop?

## 30-second application
- New request:
- Displaced item:
- Decision:
	- [ ] Yes (and accept displacement)
	- [ ] No
	- [ ] Yes, but renegotiate scope/timing

## Helpful link
Use [[Boundary Scripts]] to communicate the decision.

### Failure mode and fix
Failure mode: You say yes by default and absorb the cost silently.

Fix: practice saying the displacement out loud (even to yourself) before committing.

### Covers / Does not cover
- Covers: Making tradeoffs explicit before committing.
- Does not cover: Goal setting.

## Next action
- For the next request you receive, write the displaced item before you reply.
