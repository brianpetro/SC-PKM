> [!summary]
> GTD is a set of moves that turns vague commitments into clear next actions and trusted lists.

> [!When to use]
> When you feel mentally overloaded because tasks and commitments are living in your head.

### The 5 steps
1) **Capture**: collect everything into a trusted inbox.
2) **Clarify**: decide what each item means.
3) **Organize**: put it where you will see it at the right time.
4) **Reflect**: review so the system stays current.
5) **Engage**: do the work.

### Clarify questions that matter
- Is it actionable?
- If yes: what is the very next physical action?
- If no: trash, reference, or incubate.

### Concrete example
Inbox item: "Update starter vault reference table".
Clarify -> Next action: "Add 10 new topic notes to +reference.md".
Organize -> Put the action in your task system and keep links to the context note.

### Failure mode and fix
Failure mode: You capture but never clarify, so the inbox becomes storage.

Fix: attach a daily or weekly review habit (see [[PKM Daily Review]] and [[PKM Weekly Review]]).

### Covers / Does not cover
- Covers: The durable GTD processing moves, independent of any app.
- Does not cover: Full GTD lists and contexts implementation (see [[Getting Things Done by David Allen]] for deeper reference).
