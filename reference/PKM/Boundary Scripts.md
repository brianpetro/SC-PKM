> [!summary]
> Boundaries work when they are clear, brief, and tied to tradeoffs.

> [!When to use]
> When you feel overcommitted or you need a clean way to respond to new requests.

### Scripts
# Boundary Scripts

> [!summary]
> Clear, short scripts that protect priorities without creating drama.

## 1) No (simple)
I cannot take this on right now.

## 2) No + alternative
I cannot take this on right now. If helpful, I can point you to {reference/person}.

## 3) Yes, but later
Yes, I can do this. Earliest I can start is {date}. Does that still help?

## 4) Renegotiate scope
I can do {smaller scope} by {date}, or the full version by {later date}. Which do you want?

## 5) Priority check (forces tradeoff)
I can do this, but it will displace {current commitment}. Is that the right trade?

### Failure mode and fix
Failure mode: You give vague replies that create limbo.

Fix: use one of these scripts and include a date or a clear no.
