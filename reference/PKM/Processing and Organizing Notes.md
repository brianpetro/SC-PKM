> [!summary]
> Turn raw captures into structured notes using folders, tags, links, and rules.

> [!When to use]
> If captures never become usable notes, this note shows a triage path that turns raw input into structured artifacts, without spending hours.

### Triage steps and decision tree
- Delete.
- If yes: define the next action and put it where you execute.
- If yes: store it with enough context to retrieve later.

### Convert captures into note types or tasks
- Fleeting capture -> task (with link to context).
- Meeting capture -> meeting note -> decision log + actions.
- Source clip -> literature note -> permanent note(s).

### Apply tags, metadata, and links consistently
- Minimal metadata fields (see [[Consistency in Note Taking]]).
- 0-2 primary tags, only if they improve retrieval.
- 1-3 meaningful links that explain relationships (see [[Tag Canon]]).

### File into PARA or chosen structure
- Keep folders shallow.
- Use folder boundaries mainly for lifecycle (inbox, references, archive).
- Use project hubs and MOCs for navigation.

### Automation opportunities and guardrails
- Templates for consistent frontmatter.
- Hotkeys for capture and processing.
- Optional AI suggestions for tags and links.
