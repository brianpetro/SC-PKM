> [!summary]
> The lifecycle from capture to distillation to permanent knowledge, with clear note roles.

> [!When to use]
> If all notes blur together, this note clarifies note roles and transitions so you know what each note is for, without adding complexity.

### Define each note type and its purpose
- Fleeting capture: fast, low-friction, high volume.
- Source-bound notes: preserve context and citations from an input.
- Permanent notes: your own words, one idea per note, linkable and reusable.

### Lifecycle stages and transition rules
- If it is raw and time-sensitive, it stays in the inbox until processed.
- If it depends heavily on a source (quotes, page numbers, screenshots), it is a literature note.
- If it stands alone and can be linked out of context, it is a permanent note.

### Where metadata and tags fit per stage
- Capture: minimal metadata. Keep speed high. Add only what enables processing (source + intent).
- Literature/source: include provenance (author, link, page, timestamp), so it stays traceable.
- Permanent: prioritize title quality, links, and a short scope boundary.

### Examples of lifecycle in daily workflow
- Monday: You capture a fleeting note in the inbox: "AI control center idea: Obsidian + templates + hotkeys. Why: reduce friction for review and synthesis."
- Tuesday: You read a post and make a literature note with 2 quotes and your paraphrase.
- Wednesday: You create a permanent note: "AI as a review co-pilot" with 2 links to related concepts.

### Avoid blurred roles and duplicated content
- Enforce an inbox boundary: raw notes live in [[Capturing Information]] until processed.
- You cannot tell what is trustworthy.
- You repeatedly reread old notes because they are not distilled.
