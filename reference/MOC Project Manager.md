> [!summary]
> Map of content for project managers: meetings, decisions, tasks, and stakeholder context in a PKM system.

> [!When to use]
> If PM context is scattered across docs and chats, this map gives you a home for meetings, decisions, actions, and stakeholders, even when you juggle many threads.

### Reading path
- [[PKM in Project Management]] -> what belongs in the system
- [[Meeting Notes Workflow]] -> extract outcomes, not transcripts
- [[Decision Logs]] -> preserve rationale and avoid re-deciding
- [[PKM Weekly Review]] -> weekly reset that prevents drift
- [[Time Management]] -> protect maker time and follow-through
- [[Processing and Organizing Notes]] -> keep capture and follow-ups clean
- [[Efficient Retrieval]] -> find the right decision or action in seconds
- [[Queries and Saved Searches]] -> queues for actions, decisions, risks, and stale projects

### PM information flows and outcomes (alignment, delivery, visibility)
- Meetings -> decisions -> actions -> status.
- Stakeholder context -> tradeoffs -> rationale.
- Risks -> mitigations -> follow-ups.
- High context switching.

### Core artifacts (project hubs, meeting notes, decisions, actions, risks)
- **Meeting notes** that extract decisions and actions (see [[Meeting Notes Workflow]]).
- **Decision log** entries for meaningful decisions (see [[Decision Logs]]).
- A simple risk list (inside the hub or as linked notes).

### Weekly rhythm (review, plan, update stakeholders)
- Fix: write updates by pulling decisions and actions from linked meeting notes.
- what changed,
- what is next,
- what is blocked.

### Linking patterns (projects, stakeholders, dependencies, status)
- Meeting note -> project hub (always).
- Decision log -> project hub (always).
- Action item -> project hub (when it affects delivery).
- Stakeholder note -> project hub (for recurring stakeholders).

### Failure modes (meeting note dumps, action loss) and fixes
- Fix: use [[Meeting Notes Workflow]] and always extract decisions/actions.
- Fix: tie actions to calendar blocks via [[Time Management]].
- Fix: use [[Decision Logs]] and link to evidence and meetings.
- Fix: protect a recurring [[PKM Weekly Review]] slot.
