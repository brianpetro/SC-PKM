> [!summary]
> Step-by-step process to combine notes into a new insight, framework, or draft.

> [!When to use]
> If you have notes but no draft, this workflow shows how to combine a small cluster into an outline or artifact, even if the cluster is messy.
> If you keep summarizing but never creating, this note shows the core synthesis moves that produce new insight, without fake synthesis.

### Select a cluster of related notes to combine
- Pick one question you want to answer.
- Gather 5-10 notes that speak to that question.
- Add them to a temporary working list (or a scratch note).

### Review and extract key claims and tensions
- Skim notes using summaries and highlights (see [[Progressive Summarization]]).
- Extract:
- 3-7 key claims,

### Build an outline or argument map
- Outline: headings that represent claims/categories.
- Argument map: claim -> evidence -> counterargument.
- Framework: categories + examples under each.

### Define each synthesis method clearly
- Draw relationships between concepts (see [[Mind Mapping]]).
- Summary:
- Compress one source or cluster into the key points.

### When to use each method in practice
- Summary when you need a fast brief.
- Compare/contrast when choosing between approaches.
- Framework building when you keep collecting notes but lack structure.

### Prompts and questions that drive synthesis
- "What are the 3 claims that show up across these notes?"
- "Where do these notes disagree, and why?"
- "What is the smallest set of categories that explains all examples?"

### Examples of method outputs (frameworks, models)
- A 2x2 that compares two workflows on speed vs reliability.
- A checklist distilled from repeated failure modes.
- A decision memo that records options, constraints, and rationale.

### Draft the synthesized artifact and link sources
- Write the first pass fast.
- Under each section, link the source notes you used.
- Turn repeated source ideas into evergreen candidates.

### Avoid fake synthesis (just restating sources)
- Failure mode: "I rewrote the same notes in a new order." Fix: synthesis requires a new structure (categories, tradeoffs, claims, or a decision).
- Failure mode: "I collected 30 notes and never drafted anything." Fix: timebox synthesis and produce a small artifact first.

### Publish, refine, and feed back into evergreen notes
- Create or update one [[Evergreen Notes]] note that captures the synthesized insight.
- Update 1-3 source notes with the clarified phrasing.
- Add the artifact link to your relevant MOC or project note.
