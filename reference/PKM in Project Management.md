> [!summary]
> Apply PKM to tasks, milestones, decisions, references, and coordination.

> [!When to use]
> If project info is scattered and hard to reuse, this note shows how to store and link the artifacts that keep delivery on track, without adopting a new tool.

### Project artifacts PKM should store and link
- Project hub: goals, scope, status, key links.
- Stakeholders: who cares, what success means for them.
- Decisions: rationale and tradeoffs.

### Task and milestone tracking patterns
- Copying your entire task tracker into the vault. Fix: store the context in the vault and link to the tracker for execution details.
- Every project has a hub note that links to key tasks and milestones.
- Tasks reference the note that contains the rationale (decision, meeting outcome).

### Meeting notes into decisions and actions
- Meeting note dumps with no actions. Fix: enforce an "Outcomes" section and process it immediately after the meeting.
- extract decisions,
- extract actions with owners,

### Project hub note structure and templates
- Objective / outcome statement
- Scope (in/out)
- Current status (one paragraph)

### Review cadence and stakeholder visibility
- Use [[Time Management]] to protect time for the review and update loops.
- Daily: capture key outcomes and keep inbox clean.
- Weekly: update project hubs, review risks, prepare stakeholder update.
