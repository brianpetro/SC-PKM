> [!summary]
> Courses, workshops, and tutorials to learn PKM methods and tools.

> [!When to use]
> If you keep collecting PKM courses instead of practicing, this note helps you choose one and turn it into vault assets, without binge-watching.

### Course types and what they teach
- Framework courses (PARA, Zettelkasten, LYT-style navigation).
- Tool courses (Obsidian workflows, plugin setups, templating).
- Output courses (writing pipelines, synthesis, publishing).

### Criteria for choosing a course or workshop
- Does it match your goal in the next 30 days?
- Does it include practice assignments?
- Does it teach a durable method (not just a tool)?

### Learning plan and practice assignments
- Week 1: set up capture + inbox + one review cadence.
- Week 2: practice distillation (1 source -> 3 atomic notes).
- Week 3: practice retrieval (MOCs + saved searches).

### Communities and accountability options
- Study group with a weekly check-in.
- Public learning log (one paragraph per week).
- Pair with a friend: trade artifacts and feedback.

### How to turn course notes into vault assets
- Course notes become valuable when they become reusable assets.
- Conversion pattern: 1) Capture course notes as raw notes.
- 2) Extract 3-10 atomic notes (principles, methods, checklists).
