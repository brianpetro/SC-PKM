> [!summary]
> Review at expanding intervals to strengthen memory and long-term retention.

> [!When to use]
> If you understand today and forget next week, this note shows how to use spaced repetition so learning sticks, without overloading your queue.

### Core principle and spacing effect basics
- Memory decays unless you retrieve it.
- Retrieval strengthens the memory trace.
- Increasing intervals forces effortful recall, which is what improves retention.

### What content is worth spaced review
- Concepts you want to use repeatedly.
- Vocabulary, formulas, or definitions you must recall quickly.
- Mental models that change decisions.

### Flashcard and prompt formats that work
- Q/A: "What is X?" -> "X is ..."
- Cloze: "X is a tradeoff between __ and __."
- Scenario: "Given situation Y, what would you do and why?"

### Scheduling and review cadence options
- Fix: cap new cards per week and archive low-value prompts.
- Manual: review a small set during [[PKM Daily Review]] or weekly review.
- Tool-assisted: a spaced repetition app with a capped daily queue.

### Pitfalls like overloading the queue
- Fix: cap daily reviews and only add prompts for concepts you truly want to retain.
- Fix: write prompts that require explanation or application.
- Fix: link prompts to context and application notes; students should start at [[MOC Student]] for a full workflow.
