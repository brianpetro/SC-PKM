> [!summary]
> Index of reusable reference material and knowledge in a PARA-aligned system.

> [!When to use]
> If References becomes a junk drawer, this note helps you store reference material so it stays useful, without burying what matters.

### Purpose and scope
- References are reference material you may reuse: frameworks, reading notes, checklists, examples, snippets.
- If you might cite it or reuse it later, it belongs here; if you are actively delivering, it belongs in [[Projects]].
- Keep References easy to search and easy to discard; most references are disposable.

### How to use this MOC
- Prefer small, topic-based hubs (MOCs) over deep folders; link out to the best notes.
- Distill long sources over time (see [[Progressive Summarization]]) so reuse gets faster.
- Archive outdated references to [[Archives]] instead of keeping them mixed with current references.

### Key links
- [[Knowledge Sources]] keeps provenance so references stay trustworthy.
- [[Progressive Summarization]] makes long sources scannable when you return later.
- [[PKM References]] helps you curate what is worth keeping in References in the first place.

### Maintenance and review
- Monthly: prune duplicates and outdated references; keep only what you actually use.
- When you use a reference, improve it immediately (add an example, link, or summary).

### Common pitfalls
- Turning References into a dump - if you will not reuse it, delete or archive it.
- Keeping raw highlights forever - distill or convert the best parts into [[Permanent Notes]].
