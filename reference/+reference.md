Reference notes live here. They explain concepts, strategies, and serve as reference material.
## WDLL
- Clear guide for the creation of reference notes

## HOW
Each reference note is a small guide that:
- delivers a visible win fast (aim: <2 minutes to understand and apply)
- names when to use it (the friction it fixes)
- ends with exactly one next action that creates or upgrades an artifact

Frontmatter (minimal, consistent):
- `WHAT`: the note's promise (1 sentence)
- `aliases`: search terms only (0-3)
- `tags`: 0-4 canonical tags (see [[Tag Canon]])

Body (current standard shape):
- Summary callout (1 line)
- "When to use" callout (friction -> win, with a constraint)
- Outline (the 5 bullets)
- Tight sections expanding the outline (include 1 concrete example + 1 failure mode/fix when it fits)
- Covers / Does not cover (1 line each)
- Next action (exactly one)

Navigation + durability rules (kept lightweight):
- Links should be earned in sentences (avoid naked link blocks).
- Avoid version-sensitive UI strings; describe the intent of the command/action.

Formatting:
- Utilize callouts to enable quick scan of essential details.
