> [!summary]
> When everything is a priority, nothing is. This board forces choices.

> [!When to use]
> When your list is overloaded or you keep reacting to urgent requests.

### Template
```
# Priority Board

> [!summary]
> Sort work by urgency and importance, then choose what gets time.

## Eisenhower grid
| | Urgent | Not urgent |
| --- | --- | --- |
| Important | Do / schedule | Plan (time block) |
| Not important | Delegate | Delete |

## Quick rules
- Important + not urgent is where leverage lives.
- If it is urgent but not important, try to delegate or constrain.

## Pick 3
- [ ] One Do now
- [ ] One Schedule
- [ ] One Delete or Delegate
```

### Failure mode and fix
Failure mode: You label everything as important.

Fix: define 'important' as 'moves a goal or project outcome' (see [[Focus on Relevance]]).

### Covers / Does not cover
- Covers: Quick prioritization using a 2x2 grid.
- Does not cover: Detailed project planning.

## Next action
- Fill this board with 10 items and choose one Do now, one Schedule, and one Delete/Delegate.
