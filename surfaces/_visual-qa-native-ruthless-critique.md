## Ruthless critique

- The native set is disciplined but shallow: one screenshot per surface prevents bloat, but it leaves no format variants and little room to compare raw vs crop.
- The strongest native screenshots are the ones with real PKM context visible: editor, file explorer, search, graph, backlinks, canvas, bases, properties, and page preview.
- The weakest native screenshots are mostly crop candidates, not failures: status bar, ribbon, tabs, settings, community plugins, audio recorder, Publish, Sync, and Web viewer need tighter crops or more truthful disabled-state framing.
- Native surfaces should not compete with plugin surfaces for campaign attention. Their role is baseline context: show what Obsidian provides before Smart Plugins add retrieval, graph, context, and synthesis.
- The next native 100X pass should recapture the 14 crop candidates as focused crops and create a separate campaign shortlist instead of adding variants to every surface note.
