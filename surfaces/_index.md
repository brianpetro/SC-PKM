# Surfaces screenshot index

> [!summary]
> This folder is the screenshot index: one note per UI surface, with indexed screenshots for that surface and a How section for producing the next capture.

## WHY

Surface notes prevent screenshot evidence from being trapped in one campaign, run folder, or archive path. Scenes still own proof contracts; surfaces own screenshot buckets.

## WDLL

```md
Every plugin and native Obsidian surface has a well-named note in surfaces/. Each note contains indexed screenshots for that exact surface, capture instructions, and backlinks to screenshot-production policy.
```

## How to use

1. Pick the surface note before opening Obsidian.
2. Read the How section and capture checklist.
3. Preserve a raw full-window screenshot first.
4. Add every approved raw, candidate, final, or publication screenshot back to the same surface note.
5. If a screenshot proves a workflow, link it from the relevant scene note too.

## Coverage constraints

- Use Pro naming wherever tier naming appears.
- Settings and store screenshots are support evidence unless a scene explicitly needs them.
- Native Obsidian screenshots should be fixture-safe baselines, not accidental captures of private vault state.
- When a surface has no screenshot yet, the note is still useful as the capture contract and future bucket.
