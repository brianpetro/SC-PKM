# Prompt Change Incident Review

A prompt change incident review explains what changed, how the behavior shifted, who noticed, and what guardrails should prevent repetition. In AI systems, small changes to wording or tool instructions can produce large workflow differences.

The review should separate symptoms from causes. A user might report that the assistant became less helpful, but the underlying issue could be missing context, an over-constrained instruction, or a model routing change. The incident note should link to evaluation cases that would have caught the regression.

Incident reviews become valuable when they feed back into the AI Ops Research Hub. They should create new tests, update launch gates, and clarify which product promises require special protection.
