# Support Patterns for AI Failures

Support patterns reveal where AI features fail in real customer workflows. Common patterns include hallucinated details, incomplete actions, brittle tool use, missing context, unexpected refusals, and unclear recovery paths.

Support notes should not only list bugs. They should describe the user expectation, the system behavior, the severity, and whether the issue belongs in product design, prompt design, model evaluation, or customer education. This makes support evidence useful for roadmap and launch decisions.

When connected to the AI Ops Research Hub, support patterns can become evaluation cases, incident triggers, or product constraints. A recurring failure mode should not stay trapped in the support queue.
